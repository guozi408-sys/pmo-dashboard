概要

本資料は、NSS社内タスク情報管理用のExcelであり、京成電鉄案件に関する内部タスク、担当、ステータス、確認事項を管理するための台帳である。

主要事項

  • 社内タスクの担当者、ステータス、期限を管理する用途の資料。
  • PMO dashboard では action-log、issue-log、risk-log への反映候補になる。
  • 顧客確認事項と社内対応事項を分離して扱う必要がある。

確認事項候補

  • 各タスクの正式 owner、due_date、顧客依存有無の整理。
    NSS / SHEET-001

リスク候補

  • 社内タスクと顧客依頼事項が混在すると、進捗報告や責任分界が不明確になる可能性。
    medium / SHEET-001

アクション候補

  • タスク情報をaction-logへ整理し、顧客依頼事項と社内対応事項を分ける。
