概要

本資料は、NSS社内向けのプロジェクトメモ資料であり、京成電鉄向けWeChatミニプログラム案件の社内コミュニケーション、システム構成、情報設計、性能、セキュリティ、リスクに関する内部検討内容を整理している。顧客向けではなく、PM・技術チーム間の認識合わせに用いる資料である。

主要事項

  • 顧客向け説明で避けるべき表現や社内コミュニケーションの分担が整理されている。
  • WeChatミニプログラム、中間サーバ、既存システム、WeChat Pay、顔認証のデータ連携イメージが検討されている。
  • OpenID/UnionID、個人情報、マーケティングデータ、越境データ、WeChat規約などの情報設計課題がある。
  • 性能面では中国-日本間通信、APIレスポンス、キャッシュ、ピーク時乗車集中が検討対象。

判断候補

  • 顧客向けに「AIで開発」「横展開」等の表現を避ける。
    candidate / SLIDE-001

確認事項候補

  • 管理画面を誰が何のために使うか、必要機能とデータ再利用範囲。
    NSS / 京成電鉄 / SLIDE-005
  • WeChatミニプログラムで取得・管理できる個人情報、行動ログ、マーケティングデータの範囲。
    NSS / 京成電鉄 / SLIDE-007
  • 本番環境の冗長構成、バックアップ、リストア、性能目標、SLO/SLA。
    NSS / 京成電鉄 / SLIDE-011

リスク候補

  • 個人情報・顔データ・WeChat規約により、想定データ連携やマーケティング利用が制約される可能性。
    high / SLIDE-009
  • 中国-日本間通信やAPIコール遅延により、ユーザー体験や改札運用に影響が出る可能性。
    medium / SLIDE-014
  • 性能・セキュリティ・運用要件が未定義のままだと、設計・見積・テスト範囲が不安定になる可能性。
    medium / SLIDE-018

アクション候補

  • 管理画面、情報設計、性能、セキュリティ、運用保守の確認観点を要件定義に反映する。
    candidate / NSS PM / Tech Lead / SLIDE-010
