REL-001 リリース計画

1. 文書概要

本文書の目的

本書は、KEISEI WeChatミニプログラムの段階リリースに必要な対象範囲、判定条件、事前確認、作業、切戻し、運用引継ぎおよび確認事項を整理し、顧客・関連ベンダー・NSS間のリリース計画レビューに用いるものである。

  • 対象プロジェクト: KEISEI WeChat Mini Program Project
  • 前提資料: MGT-001、REQ-001、ARC-001、API-001、WBS-001、SCR-001、TST-001および登録済みsource summary
  • 文書ステータス: review_required
  • SRC-011: 他案件の様式参考であり、KEISEIのリリース事実には使用しない

2. リリース方針

  • Ph.1: WeChatミニプログラムによる現行eTicket相当の購入・決済・チケット表示を中心とする候補。
  • Ph.1.1: 顔認証用写真撮影・アップロード等を独立段階とする候補。Ph.1との統合可否は未確定。
  • Ph.2: QR乗車券、座席指定、乗車QR発券等を2027年4月以降の候補として整理する。
  • 段階リリースとし、各フェーズのscope、外部I/F、試験、審査、運用準備が整った範囲のみを対象候補とする。
  • 既存システム側の改修実装、正式scope外の周辺サービス、未合意の管理・マーケティング機能は対象外候補とする。

Ph.1の資料上の候補日は2027年1月から3月中旬まで複数の口径があり、正式日ではない。顧客承認、外部接続、UATおよびWeChat審査を踏まえて確定する。

5. リリース前チェック

  • MGT、REQ、ARC、API、WBS、SCR、TSTおよび本書のレビュー状態を確認する。
  • 本番環境、ネットワーク、アカウント、証明書、秘密情報、設定差分、監視を確認する。
  • 外部APIのendpoint、認証、接続元制限、状態・エラー、疎通結果を確認する。
  • WeChat Mini Programの審査、公開条件、権限、加盟店、バージョンを確認する。
  • 個人情報・顔情報の同意、保存、削除、マスキング、監査、越境条件を確認する。
  • 管理画面の利用者、権限、初期アカウント、操作ログを確認する。
  • 運用連絡体制、障害窓口、外部ベンダー連絡先、エスカレーションを確認する。
  • リリース手順、切戻し手順、データ確認、チェックリスト、証跡保管先をレビューする。

6. リリース作業

事前準備

対象バージョン、変更一覧、環境差分、設定値、アカウント、証明書、データ、監視、連絡先、作業参加者を確認する。

手順候補

  1. Go/No-Go判定と作業開始連絡
  2. 対象環境・外部接続・監視状態の事前確認
  3. 中間サーバ / 管理機能のデプロイ候補
  4. Mini Programの公開・設定反映候補
  5. 外部API疎通、決済、購入、表示等のスモーク確認
  6. 監視・ログ・状態整合の確認
  7. 顧客確認と作業完了連絡

具体的なコマンド、順序、作業時間帯、停止有無、担当者、チェックポイントは環境・リリース方式確定後に作成する。上記は確定手順ではない。

7. ロールバック / 切戻し方針

  • 切戻し判断は重大障害、購入・決済・発券不能、状態不整合、個人情報・顔情報事故、復旧見込み等を基準候補とする。
  • アプリケーション、設定、Mini Program公開状態、外部接続、データ変更を分けて切戻し可否を評価する。
  • 外部システムで既に成立した決済・発券・顔登録等は単純なバージョン切戻しだけでは解消しないため、照合・補正・有人対応を準備する。
  • 顧客、運用担当、外部ベンダーへの連絡と利用者案内の判断者を事前に確定する。
  • 切戻し後は状態整合、監視、再開条件、再リリース計画を確認する。

実際の切戻し手順、復旧目標、データ補正権限、許容停止時間は未確定である。
