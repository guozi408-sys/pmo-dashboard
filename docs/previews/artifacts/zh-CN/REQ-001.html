6. 正文

6.1 文档概要

本文档针对京成电铁 WeChat Mini Program，整理业务需求、功能需求、外部系统联动、非功能需求以及确认事项，作为相关方评审和后续设计的输入。

对象是访日中国游客在 WeChat 上购买和支付 Skyliner 相关乘车券，并通过人脸认证或未来的乘车用 QR 乘车的服务。包含中间服务器、管理功能以及既有系统联动。

本文档状态为 review_required。需求、责任边界、API 规格、上线日期、UAT 条件仍有未确定事项，并非 approved 文档。

6.2 项目背景

背景与课题

  • 面向访日中国游客，需要无需安装额外 App 的购买和乘车体验。
  • 设想在利用既有乘车券管理、人脸认证、检票、座席管理等的同时，新建 WeChat Mini Program 和中间服务器。
  • 外部 I/F 的规格和提供日期、数据处理、供应商责任边界尚未确定，对设计、估算和日程产生影响。

目的

  • 建立可在 WeChat 内完成乘车券检索、购买、支付、使用状态确认的流程。
  • 分阶段构建可支持人脸认证以及后续乘车用 QR 的业务和系统基础。
  • 作为京成电铁的官方数字触点，使其能够管理运营所需的订单、状态和异常信息。

对象用户

  • 主要使用简体中文的访日中国游客
  • 京成电铁的车站工作人员、业务管理员、咨询应对负责人
  • NSS 以及外部系统供应商的运营和维护负责人

6.3 对象范围

范围内

  • 通过 WeChat Mini Program 选择、购买 Skyliner 相关乘车券并进行 WeChat Pay 支付
  • 购买后的人脸注册链接联动、人脸认证结果和订单状态显示候选
  • 简体中文 UI、主要画面和画面跳转、面向用户的错误和提示显示
  • 中间服务器、管理功能、开发/验证/生产环境
  • 乘车券管理、人脸认证、检票/打印等外部 I/F 整理和集成测试支援

范围外

  • 通过 QR 检票机正式进出站、正式发行乘车用 QR
  • 座席指定、座席/列车变更等座席相关例外处理
  • 既有系统的大规模改修
  • 高级客户分析、广告、营销自动化
  • 正式运营维护合同以及简体中文以外的多语言展开

Pending scope

  • 人脸认证未完成、失败、系统故障时的人工应对与通常流程边界
  • 取消/退款的自动处理、状态反映、手续费以及车站工作人员操作
  • 管理画面的使用者、必要功能、显示数据、营销使用范围
  • 乘车用 QR、QR 检票、座席指定的对象阶段
  • 非功能、安全、监控、运营维护、SLA/SLO

  • 取消、退款、交换、人工应对的业务规则和实施主体
  • 车站工作人员参照/操作的信息和审计记录
  • 乘车券管理侧可提供的状态查询/更新 I/F

6.5 功能需求

WeChat Mini Program 功能

  • 乘车券信息/购买入口、商品选择、订单确认、支付开始
  • WeChat Pay 结果以及订单状态显示
  • 连接到人脸注册 URL，认证状态、错误、重试提示
  • 可使用的车票、订单详情、使用提示的显示
  • WeChat 审核所需的使用条款、隐私说明、同意流程

中间服务器功能

  • 面向 Mini Program 的 API 与外部 I/F 的中继
  • 订单、支付、票券、认证状态的一致性管理候选
  • API 认证、错误处理、超时、重试、带关联 ID 的日志
  • 按环境管理设置以及外部连接目标

管理画面功能

  • 订单、支付、票券、人脸认证、异常状态的检索和参照
  • 确认咨询和人工应对所需的履历
  • 显示运营监控所需的最小状态和错误信息
  • 使用者、权限、操作履历、是否需要数据导出作为确认事项。

外部系统联动和审核对应

  • 与乘车券管理、人脸认证、WeChat Pay、检票/打印等的联动
  • 对 WeChat Mini Program 以及 WeChat Pay 的审核资料、类别、外部跳转条件的对应
  • 在设计前确认人脸照片、外部 URL、支付功能相关审核限制。

6.9 风险和课题

主要风险

  • API 规格和数据定义不足或提供延迟，导致需求、设计、估算、测试日程变动。
  • 人脸信息、个人信息、跨境数据、WeChat 规则可能限制联动方式。
  • 中日间通信、外部 API 响应、峰值使用可能影响用户体验和车站运营。
  • 性能、安全、运营需求未定义就进入后续工程的可能性。
  • Ph.1 / Ph.1.1 / Ph.2 的范围、概算估算、上线目标未确定。

应对方针

  • 优先接收现行 I/F 规格，分阶段实施 QA 和影响分析。
  • 联动更新 scope、API、architecture、WBS，并保留变更履历。
  • 对个人信息和人脸信息进行最小化、同意、保存/删除、权限、跨境条件评审。
  • 未确定项目保持 open / candidate / review_required，不在没有确认记录的情况下视为已解决。
