WBS-001 WBS / スケジュール

本書は登録済み source summary、MGT-001、REQ-001、ARC-001、API-001 および control logs に基づく計画レビュー用初版である。未来日付と作業期間は候補または確認待ちであり、顧客承認済み日程やリリース確約を意味しない。

1. 文書概要

本文書の目的

KEISEI WeChat Mini Program Project の作業、成果物、外部依存、マイルストーンおよび確認事項を一つの計画ビューに整理し、優先順位、担当候補、前後関係と日程リスクをレビューする。

対象プロジェクト・前提資料

  • MGT-001 プロジェクト計画書 v0.1
  • REQ-001 要件定義書 v0.1
  • ARC-001 システム構成定義 v0.1
  • API-001 API確認表 v0.1
  • 本書 frontmatter に記載した KEISEI 事実資料と control logs
  • SRC-011 は他案件の様式参考であり、KEISEI の日程事実には使用しない。

本文書のステータス

review_required。要件、API、責任分界、環境、テスト条件および正式リリース日が未確定のため、ベースライン確定前の WBS である。

2. スケジュール前提

対象フェーズ

  • Ph.1: 購入・決済、顔登録・顔認証、入場・紙媒体、管理機能を中心とする候補。
  • Ph.1.1: 顔写真撮影・アップロード等を Ph.1 と分離する場合の追加段階候補。
  • Ph.2: 乗車用 QR、QR 改札、座席指定等の後続段階候補。

確認済み資料上の事実

  • 初回 Kickoff 資料日: 2026-05-21（SRC-001）
  • 第2回 Kickoff / 議事メモ日: 2026-07-07（SRC-002、SRC-003）
  • 現時点では API 仕様受領、要件確定、外部結合、UAT、リリースの正式日を統一する必要がある。

未確定事項・制約

  • 初期資料の「2026年中」「2027年4月以降」と、後続資料の「2027年1～3月」には口径差がある。
  • API 仕様書、外部ベンダー改修、接続環境、WeChat 審査、個人情報確認がクリティカルな外部依存である。
  • 要件定義完了前に開発・試験・リリース日を確定しない。

5. フェーズ別 WBS

Ph.1

  1. scope・業務フロー・例外処理をレビューする。
  2. 乗車券、決済、顔認証、検札・印刷の I/F を確認する。
  3. Mini Program、中間サーバ、管理機能を設計・開発する。
  4. 内部結合、外部接続、UAT、審査、リリース準備を実施する。

Ph.1.1

顔写真撮影・アップロード等を独立段階とする場合のみ追加 WBS を設定する。Ph.1 への統合可否、追加審査、個人情報・試験影響を確認する。

Ph.2 / Pending scope

乗車用 QR、QR 改札、座席指定は後続候補とし、I/F、既存改修、対象駅・票種、費用・日程の合意後に WBS を具体化する。

フェーズ間依存関係

Ph.1 のデータ・状態モデルと外部 I/F は、Ph.2 で QR・座席を追加できる境界を残す。ただし後続機能を Ph.1 の開始条件にはしない。
