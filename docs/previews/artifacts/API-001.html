API-001 API確認表

本書は登録済み source summary、MGT-001、REQ-001、ARC-001 および ACAP METHOD-001 / METHOD-006 に基づく API・I/F レビュー用初版である。記載する API 名称は確認対象を識別するための候補名であり、正式 endpoint、項目、認証方式または実装承認を意味しない。

1. 文書概要

本文書の目的

WeChat ミニプログラム、中間サーバ、決済、乗車券管理、顔認証、検札・改札、座席管理および管理機能間の API / I/F 候補を一覧化し、顧客・外部ベンダー・NSS が確認すべき仕様、責任分界および未確定事項を管理する。

対象プロジェクト・前提資料

  • KEISEI WeChat Mini Program Project
  • MGT-001 プロジェクト計画書 v0.1
  • REQ-001 要件定義書 v0.1
  • ARC-001 システム構成定義 v0.1
  • 本書 frontmatter に記載した KEISEI 事実資料
  • ACAP METHOD-001 および METHOD-006
  • SRC-011 は他案件の様式参考であり、KEISEI の API 事実には使用しない。

本文書のステータス

review_required。正式 I/F 仕様書、接続情報、データ定義、認証、エラーおよび環境条件を受領するまで、本書は API 確認台帳であり API 仕様書ではない。

5. データ項目確認

  • チケット購入: 商品、票種、利用日、区間、人数、注文・票券 ID、状態
  • 決済: 取引 ID、金額、通貨、結果、返金、照合状態
  • QR表示 / QR乗車券: 用途、内容、署名、有効期限、検証・失効状態
  • 顔認証: 登録 URL、認証状態、結果、エラー。写真本体の流通・保存は未確認
  • 座席指定: 列車、区間、座席、在庫、変更・取消状態
  • ユーザー識別: WeChat 識別子、連絡先等の必要性と最小取得範囲
  • 管理機能: 注文、決済、認証、異常、人工対応、操作履歴
  • 個人情報 / 顔情報: 取得目的、同意、正本、保存場所、期間、削除、越境移転

項目名、型、桁数および必須条件は正式 I/F 仕様受領後に追記する。

6. 認証・認可

  • WeChat 側のログイン・ユーザー識別方式と中間サーバへの伝達範囲
  • 管理画面の認証、ロール、最小権限、セッション、操作監査
  • 外部システム接続の API key、token、署名、証明書、IP 制限候補
  • 秘密情報の発行、保管、更新、失効および環境分離
  • 顔情報・決済・個人情報へのアクセス権と委託先責任

具体的な認証方式は source で未確定であり、本版では決定しない。

10. リスク・課題

  • I/F 仕様書と接続情報の提供遅延により、設計・見積・結合試験が遅れる可能性
  • 外部ベンダー間で API と障害対応の責任分界が曖昧になる可能性
  • 認証、秘密情報、権限、ログ、監査要件が未定義であることによるセキュリティリスク
  • 顔情報・個人情報の保存、削除、同意、越境移転に関する制約
  • 決済、出票、認証、入場の状態不整合と障害時リカバリの未確定
  • 中国-日本間通信および外部 API 応答が性能・可用性へ影響する可能性

参照: RISK-001、RISK-002、RISK-004、RISK-005、RISK-007、RISK-008、RISK-013、RISK-014、RISK-015

11. 確認事項

顧客確認事項

  • API 対象範囲、優先順位、業務状態、取消・返金、人工対応
  • 管理機能、個人情報・顔情報、監査、運用保守の要件
  • 外部ベンダー回答の承認者、UAT 条件、リリース判断

外部ベンダー確認事項

  • 正式 I/F 仕様書、システム概要図、endpoint、項目、認証、エラー、環境
  • 購入情報以外の状態照会、取消・返金、認証結果、人工対応状態の連携可否
  • 接続試験、試験データ、障害連絡、保守・変更通知

NSS内部確認事項

  • 中間サーバの API 分割、状態管理、正本、冪等・再送・補正方式
  • API QA、モック、結合試験、監視・ログ、セキュリティ設計
  • 外部 I/F 不足時の代替運用と見積・日程影響
  • METHOD-001 / METHOD-006 の候補を正式 API 仕様へ反映する条件
